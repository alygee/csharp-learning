# csharp-learning

##### День 1–10: База C#

###### День 1

- Установить .NET SDK, IDE (Rider/VS Code/Visual Studio).
- Команды: dotnet --info, dotnet new console, dotnet run.
- Практика: “Hello, C#”, вывод переменных.

###### День 2

- Типы данных, var, nullable, преобразования типов.
- Практика: программа-конвертер (валюты/температуры).

###### День 3

- Условия: if, switch, тернарный оператор.
- Практика: калькулятор с меню операций.

###### День 4

- циклы (`for`, `while`, `foreach`)

###### День 5

- Методы, параметры, ref/out, перегрузка.
- Практика: вынести калькулятор в методы.

###### День 6

- Массивы, List<T>, Dictionary<TKey, TValue>.
- Практика: учет книг (добавить/удалить/найти).

###### День 7

- Строки, StringBuilder, работа с датой и временем.
- Практика: парсер строки заказа.

###### День 8

- ООП: классы, свойства, конструкторы, инкапсуляция.
- Практика: User, Product, Order.

###### День 9

- Наследование, интерфейсы, полиморфизм.
- Практика: система оплат (ICheckout, CardPayment, CashPayment).

###### День 10

- Обработка исключений, try/catch/finally.
- Практика: безопасный ввод и логирование ошибок в файл.

##### День 11–20: Продвинутый C# + мини-проект

###### День 11

- LINQ: Where, Select, OrderBy, GroupBy.
- Практика: аналитика списка заказов.

###### День 12

- record, enum, struct, init.
- Практика: модель домена для задач (TaskItem).

###### День 13

- Файлы и JSON (System.Text.Json).
- Практика: сохранение/загрузка списка задач в tasks.json.

###### День 14

- async/await, Task, отмена через CancellationToken.
- Практика: имитация внешнего API + таймауты.

###### День 15

- Делегаты, события, лямбды.
- Практика: событие “задача завершена”.

###### День 16

- Generics, ограничения типов.
  Практика: универсальный репозиторий в памяти.

###### День 17

- SOLID (базово), разделение ответственности.
- Практика: рефакторинг мини-проекта.

###### День 18

- Модульное тестирование с xUnit.
- Практика: 8–10 тестов для бизнес-логики.

###### День 19

- Повторение и устранение пробелов.
- Практика: чистка кода, нейминг, структура папок.

###### День 20

- Мини-проект #1 (консоль): Task Tracker
- CRUD задач
- хранение в JSON
- фильтрация по статусу/дате
- базовые тесты

##### День 21–30: ASP.NET Core Web API

###### День 21

- Что такое ASP.NET Core, middleware, DI, контроллеры.
- Практика: dotnet new webapi, запустить Swagger.

###### День 22

- Роутинг, DTO, валидация моделей.
- Практика: endpoint’ы для заметок (/notes).

###### День 23

- Слои: Controller → Service → Repository.
- Практика: вынести бизнес-логику в сервисы.

###### День 24

- EF Core, DbContext, миграции.
- Практика: подключить PostgreSQL или SQL Server.

###### День 25

- CRUD с БД, асинхронные запросы.
- Практика: полноценные операции create/read/update/delete.

###### День 26

- Аутентификация JWT (базово).
- Практика: регистрация/логин, защищенный endpoint.

###### День 27

- Логирование, конфигурация (appsettings), секреты.
- Практика: корректные логи и обработка ошибок.

###### День 28

- Интеграционные тесты API (минимально).
- Практика: 3–5 тестов на ключевые endpoint’ы.

###### День 29

- Docker basics для .NET приложения.
- Практика: Dockerfile + запуск контейнера.

###### День 30

- Финал мини-проекта #2: Notes API

##### Навыки:

- уверенно писать консольные программы на C#;
- использовать LINQ, async/await, исключения;
- создать Web API на ASP.NET Core;
- подключить БД через EF Core и сделать миграции;
- покрыть базовую логику тестами;
- запустить проект локально и в Docker.
